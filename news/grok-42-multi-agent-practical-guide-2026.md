## 三、实战场景1：深度市场调研

### 场景说明

你需要调研一个新赛道（比如AI硬件、新能源储能），要求产出一份有数据、有分析、有结论的调研报告。

### 推荐Prompt模板

```
我需要一份关于[行业/赛道名称]的深度调研报告。请按以下要求协作完成：

1. 搜索最新的市场数据：市场规模、增长率、主要玩家、最新融资动态（Harper负责）
2. 对搜索到的数据做逻辑分析：行业驱动力、竞争格局、潜在风险（Benjamin负责）
3. 基于分析结果，提出3个有创意的市场机会点和进入策略（Lucas负责）

请用结构化格式输出，包含数据表格和关键结论。
```

### 为什么这样写Prompt？

- **明确分工**：告诉系统哪部分该谁干，四个Agent各有侧重不会抢活
- **任务拆解**：搜索→分析→策略，形成信息流水线
- **输出要求**：指定格式，避免拿到一堆散文

### 实际效果

Harper 会搜索最新行业报告和新闻，Benjamin 会对数据做因果推理和趋势判断，Lucas 会跳出常规视角提出差异化策略。最终由 Grok 整合成一份结构清晰的报告。

**相比单一AI的优势**：信息更新（Harper实时搜索）、逻辑更严谨（Benjamin交叉验证）、策略更有创意（Lucas多角度思考）。

## 四、实战场景2：竞品分析

### 推荐Prompt模板

```
请帮我做[公司A] vs [公司B] vs [公司C]的竞品分析：

第一步：搜集三家公司的最新产品功能、定价、用户评价、最近的产品更新（Harper负责）
第二步：从产品力、定价策略、市场定位、技术壁垒四个维度做对比分析，用表格呈现（Benjamin负责）
第三步：如果我要做一个同类产品，给出3个差异化切入点（Lucas负责）

请在分析中标注数据来源。
```

### 关键技巧

- 加上"标注数据来源"——让 Harper 的搜索结果可溯源
- "用表格呈现"——逼 Benjamin 输出结构化结论而不是大段文字
- "差异化切入点"——把 Lucas 的创意锚定在商业可行性上

## 五、实战场景3：技术方案评估

### 场景说明

你在做技术选型（比如数据库选型、云服务商对比、框架选择），需要综合考虑性能、成本、维护难度。

### 推荐Prompt模板

```
我正在为一个[项目类型]选择[技术方向]方案。候选方案：
- 方案A：[具体技术]
- 方案B：[具体技术]  
- 方案C：[具体技术]

请从以下维度对比：
1. 搜索每个方案的最新基准测试数据、社区活跃度、企业使用案例（Harper负责）
2. 根据搜索结果，从性能、学习曲线、运维成本、扩展性四个维度打分（1-10分），给出量化对比表（Benjamin负责）
3. 考虑我们团队[规模/技术栈/预算]的情况，推荐最适合的方案并说明理由（Lucas负责）

请在最后给出明确的推荐结论。
```

### 为什么这个场景特别适合多Agent？

技术选型需要：**最新数据（Harper）+ 量化对比（Benjamin）+ 结合实际情况的判断（Lucas）**。单一AI往往只能做到前两步，第三步（结合你的实际情况给建议）容易流于泛泛而谈。

## 六、实战场景4：内容创作流水线

### 场景说明

你是内容运营/自媒体创作者，需要高效产出有深度的长文。

### 推荐Prompt模板

```
我需要写一篇关于[主题]的深度文章，目标读者是[受众描述]，发布在[平台]。

请按以下流程协作：
1. 搜索这个主题的最新热点、读者关注的核心问题、竞品文章的内容方向（Harper负责）
2. 根据搜索结果，列出文章大纲，确保逻辑链条完整——每一节要解决读者的什么问题、为什么这个顺序（Benjamin负责）
3. 基于大纲，撰写文章初稿，要求：[风格要求，比如口语化/专业/有趣]，每一节控制在300字以内（Lucas负责）

注意：
- 文章中引用的数据必须标注来源
- 标题要有吸引力，适合[平台]的调性
- 全文控制在[X]字左右
```

### 进阶技巧：两轮迭代法

第一轮用上面的Prompt拿到初稿后，再来第二轮：

```
请对上面的文章做优化：
1. 核实文中所有数据和事实是否准确（Harper负责）
2. 检查论证逻辑是否有漏洞，补充缺失的论据（Benjamin负责）
3. 优化语言表达，让每一段的开头更有钩子感（Lucas负责）
```

两轮下来，文章质量远超一次性生成的结果。

## 十一、写好Prompt的5条黄金法则

总结上面8个场景的共同规律：

### 法则1：明确指定Agent分工

**差Prompt**：帮我分析一下特斯拉的财报。

**好Prompt**：帮我分析特斯拉2026年Q1财报。Harper搜索最新财务数据，Benjamin做财务指标分析和同比对比，Lucas给出投资观点。

### 法则2：任务按流程拆解

先搜索→再分析→最后创意/建议。这个顺序和四个Agent的能力链是匹配的。

### 法则3：指定输出格式

"用表格呈现"、"用Markdown格式"、"控制在X字以内"——明确的格式要求让输出更可用。

### 法则4：提供上下文

告诉Grok你的背景、目标受众、使用场景。上下文越充分，Agent的决策越精准。

### 法则5：善用两轮迭代

第一轮拿初稿，第二轮让Agent自我检查和优化。这比一次性追求完美高效得多。

## 十三、常见问题

### Q1：Grok 4.2和Grok 4有什么区别？

Grok 4 是 xAI 的基础大模型，单一模型架构。Grok 4.2 在 Grok 4 的基础上加入了**多智能体协作架构**，4个Agent（Grok+Harper+Benjamin+Lucas）同时工作，在复杂任务上表现更强。

### Q2：我能单独和Harper/Benjamin/Lucas对话吗？

不能单独指定某一个Agent工作。但你可以在Prompt中引导分工，让特定Agent在某个环节发挥更大作用。系统会自动根据任务性质分配Agent权重。

### Q3：四个Agent同时工作，配额消耗会翻4倍吗？

不会翻4倍，但**复杂任务确实比简单提问消耗更多配额**。建议把简单问题和复杂任务分开处理——简单问题不需要触发多Agent协作。

### Q4：免费版能用Grok 4.2吗？

不能。Grok 4.2 多智能体是 **SuperGrok 专属功能**，最低需要 SuperGrok 月会员（$30/月）。

### Q5：Grok 4.2的中文效果好吗？

日常中文对话和分析没问题。但在极其精细的中文创意写作（比如诗词、文学创作）上，建议搭配人工润色。
